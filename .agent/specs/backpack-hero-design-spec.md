## 🎯 핵심 컨셉

> **"가방 속 배치가 곧 전략이다"**

플레이어는 제한된 크기의 **가방(인벤토리)**에 아이템을 배치하고, 배치된 아이템들이 **턴제 전투**에서 자동으로 작동합니다. 아이템 간 **인접 시너지**와 **배치 전략**이 승패를 가르는 핵심입니다.

### 백팩히어로의 핵심 요소
- ✅ **그리드 기반 인벤토리**: 가방이 2D 그리드, 아이템마다 차지하는 칸이 다름
- ✅ **위치 기반 시너지**: 인접한 아이템끼리 보너스 효과 발동
- ✅ **자동 전투**: 배치된 아이템이 위에서 아래로 순서대로 작동
- ✅ **로그라이크 진행**: 전투 → 보상 선택 → 가방 정리 → 다음 전투

## 📦 인벤토리(가방) 시스템

### 가방 구조
- **초기 크기**: 4×4 그리드 (16칸)
- **확장**: 특수 아이템으로 5×5, 6×5 등으로 확장 가능
- 각 칸에 아이템의 일부가 배치됨

### 아이템 배치 규칙
- 아이템마다 **차지하는 칸 수와 모양**이 다름 (1×1, 1×2, 2×2, L자 등)
- 아이템을 **드래그 앤 드롭**으로 배치
- **회전 가능** (90도 단위) → mLua 구현 가능 여부 확인 필요
- 겹치는 배치 불가

### 인접 시너지
| 시너지 타입 | 조건 | 효과 예시 |
|------------|------|----------|
| **무기 강화** | 검 옆에 숫돌 | 공격력 +2 |
| **방어 연계** | 방패 옆에 방패 | 방어력 합산 +50% |
| **마법 증폭** | 지팡이 옆에 마나석 | 마법 데미지 ×1.5 |
| **회복 시너지** | 포션 옆에 약초 | 회복량 +3 |

## ⚔️ 전투 시스템

### 기본 구조
- **턴제**: 플레이어 턴 → 적 턴 교대
- **자동 작동**: 가방 속 아이템이 **위에서 아래, 왼쪽에서 오른쪽** 순서로 발동
- **전투 전 배치 단계**: 전투 시작 전 가방 정리 시간 제공

### 플레이어 스탯
| 스탯 | 초기값 | 설명 |
|------|--------|------|
| HP | 50 | 0이 되면 런 종료 |
| 방어력 | 0 | 턴마다 리셋, 아이템으로 획득 |
| 마나 | 3 | 특수 아이템 사용에 필요 |

### 적 행동 패턴
- 적은 다음 턴에 할 행동을 **미리 표시** (인텐트 시스템)
- 표시 예: 🗡️ 공격 8 / 🛡️ 방어 5 / 💀 강공격 15

## 🔧 mLua 기술 고려사항

> [!IMPORTANT]
> mLua(메이플스토리 월드)는 일반 Lua와 다른 독자적 API를 사용합니다. 아래 사항의 구현 가능 여부를 확인해야 합니다.

### 확인 필요 항목
1. **드래그 앤 드롭**: mLua UI에서 아이템 드래그 가능 여부 → 불가능하면 **클릭 기반 배치**로 대체
2. **2D 그리드 UI**: SpriteGUI로 그리드 구현 가능 여부
3. **CSV 데이터**: 아이템/몬스터 데이터를 CSV로 관리
4. **애니메이션**: 전투 이펙트, 데미지 표시 등
