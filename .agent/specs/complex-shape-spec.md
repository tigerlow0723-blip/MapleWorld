# 작업 명세서: 비정형(테트리미노) 아이템 배치 시스템 개편

> **목표**:
> 1. 오직 너비(width)와 높이(height)만을 가진 직사각형(Box) 기반 아이템 시스템을 폐기한다.
> 2. `T`자, `ㄱ`자, `+`자 등 비정형 형태(Tetromino)를 표현할 수 있도록, 아이템 데이터베이스 구조를 2차원 배열(마스크)로 재설계한다.
> 3. Inventory 로직의 스냅(Snap), 터치, 충돌 검출(CanPlace) 함수를 비정형 2차원 배열 마스크 기반으로 100% 개편한다.

---

## 1. 아이템 형태(Shape) 마스크 구조 설계 (데이터 테이블)

기존의 `width`, `height` 숫자 2개로는 복잡한 형태를 표현할 수 없습니다. 
각 아이템의 형태를 0(빈칸)과 1(차지함)로 이루어진 행렬(Matrix) 구조체를 뜻하는 **Shape Mask 문자열**로 관리합니다.

### 1-1. `ItemTable` 데이터 보완 (비정형화)
`Data_Table - 아이템리스트` (CSV/기획 데이터)에 신규 열(Column)을 추가합니다.
- **신규 필드**: `shapeMask` (타입: string)
- **규칙**:
  - 행(Row) 구분은 쉼표(`,`) 기호를 사용합니다.
  - 빈칸은 `0`, 아이템 실물칸은 `1`로 표기합니다.
- **사례**:
  - `단검 (1칸)`: `"1"`
  - `긴 방패 (1x2)`: `"1,1"` (세로 2칸)
  - `양손검 (2x1)`: `"11"` (가로 2칸)
  - `ㄱ자 부메랑`: `"11,10"` (가로2칸 아래에 왼쪽 1칸)
  - `T자 십자가`: `"111,010"` (가로3칸 아래 중앙 1칸)

## 2. Item.mlua (데이터 파싱 및 모양 추출)

`ItemTable`에서 불러온 `shapeMask` 문자열을 스크립트 메모리에서 읽기 쉬운 2차원 숫자 테이블(Array)로 변환해 주는 작업이 필요합니다.

### 2-1. 프로퍼티 추가
- `property table shapeMatrix = {}` : 파싱된 2차원 배열 마스크를 담을 프로퍼티 추가. (초기값 빈 테이블)
- 기존 `width`, `height` 프로퍼티는 유지합니다 (마스크 행렬의 최대 외곽 가로/세로 길이를 뜻하기도 함).

### 2-2. 메서드 개편: `InitFromTable`
- `shapeMask` 필드값을 불러옵니다. (`_DataService:GetCell(dataSet, row, "shapeMask")`)
- 불러온 문자열 마스크를 쪼개어 `self.shapeMatrix`에 2차원 테이블 형태로 파싱 및 캐싱(저장)하는 알고리즘을 추가합니다. (문자열 쉼표 파싱 및 `string.sub` 활용)

### 2-3. 메서드 개편: `GetShapeCells()`
- **기존**: `for r=0~h-1`, `for c=0~w-1`를 순회하며 모두 테이블에 넣었습니다. (=직사각형)
- **변경 후**:
  1. `self.shapeMatrix` (파싱된 행렬)의 `row`와 `col`을 순회합니다.
  2. 행렬의 값이 `1`인 경우의 좌표(상대적인 `[r, c]`)만 수집하여 반환합니다.
  3. 값이 `0`인 위치는 투명한(빈) 위치이므로 배제합니다!

## 3. Inventory.mlua (배치 충돌 판정 개편)

`InventoryUI`에서 아이템을 내려놓을 때, 가방의 어느 칸이 겹치는지 이가 빠진 칸이 어딘지 검사해야 합니다.

### 3-1. 메서드 개편: `CanPlace(integer startRow, integer startCol, table shapeCells)`
- 기존처럼 `width`, `height`만 받던 인자(파라미터)를 제거하고, 방금 위에서 파싱한 **아이템의 상대 좌표 배열(`shapeCells`)**을 인자로 받도록 뜯어고칩니다.
- **충돌 방지 로직 (루프 검사)**:
  1. `shapeCells` 배열(배치할 아이템의 블록 상대좌표 모음)을 순회(for문)합니다.
  2. 입력받은 인벤토리 기준점(`startRow`, `startCol`)에, 순회 중인 블록의 상대 좌표(`cell.row`, `cell.col`)를 더해 **실제 가방 속 절대 좌표(`absR`, `absC`)**를 구합니다.
  3. 계산된 `absR`, `absC`가:
     - 인벤토리 범위를 벗어나는가? -> `return false`
     - 현재 가방(`self.grid[absR][absC]`)의 값이 `1`(빈칸)이 아닌가?(즉, 0이거나 다른 아이템id가 있는가?) -> `return false` (겹침!)
  4. 모든 블록을 무사히 통과했다면 `return true` (배치 가능!)

### 3-2. 메서드 개편: `PlaceItem(...)` & `RemoveItem(...)`
- 이 두 개의 헬퍼 함수 역시 직사각형 구역을 순회하여 칠하고 지우던 반복문에서, 똑같이 `shapeCells`를 넘겨받아 그 형체(마스크) 대로만 가방 번호를 갱신(색칠)하고 비우도록 로직을 일대일 매칭으로 수정합니다.

## 4. InventoryUI.mlua (드래그 스냅 및 렌더링 개편)

이제 드래그 중인 엔티티(아이템 이미지)는 외곽선이 사각형이 아닙니다. 색칠 놀이를 할 때도 정확하게 이가 빠진 칸은 빼고 칠해줘야(하이라이팅) 합니다.

### 4-1. 메서드 개편: `TryDropItem` 및 연계 함수
- 인벤토리의 `CanPlace`, `PlaceItem`을 호출할 때 넘겨주던 `w`, `h` 인자들을 지우고, 타겟 아이템 컴포넌트(`itemComp`)에서 `itemComp:GetShapeCells()`를 불러와서 그 결과를 전달(`pass`)하도록 수정합니다.

### 4-2. 메서드 개편: `HighlightPlacement` (내려놓을 위치 초록/빨강색 칠하기)
- (직사각형 영역 전체를 칠하는 게 안됨)
- 앞서 넘겨받은 `shapeCells` 덩어리들을 이용해서,
- 각각의 블록 위치가 색칠 가능한지 불가능한지(빈칸인지) 여부에 따라 `self.colorCanPlace` 나 `self.colorCannotPlace` 색상으로 정밀하게 마스크 부분만 색칠해 줍니다!
- 이를 통해 유저는 "아, 이거 십자가 모양으로 초록색 영역 불빛이 들어오는구나!" 하고 직관적으로 보게 됩니다.

## 5. 아이템 회전 (Rotation) 시스템 추가

비정형 마스크가 도입됨에 따라, 아이템을 90도 단위로 회전하여 `shapeMatrix`의 모양과 실제 화면(UI) 상에서 보여지는 이미지를 돌리는 기능이 필수적입니다.

### 5-1. 회전 입력 감지 (InventoryUI.mlua / InputService)
- **발동 조건 (핵심)**: `InventoryUI`에서 현재 **아이템을 마우스 좌클릭으로 꽉 잡고(Hold) 드래그 중인 상태 (`self.dragState.isDragging == true`)** 일 때만 **제한적으로 발동**합니다.
  - ⚠️ **안전장치**: 바닥(ItemDisplayArea)에 떨어져 있거나 가방(Inventory)에 가만히 배치되어 있는 아이템에는 마우스를 단순히 올려두기만 한(Hover) 상태에서 `R`키나 우클릭을 눌러도 절대 회전하지 않습니다. 물리적으로 "집어 든 채로 마우스를 누르고 있는 상태"에서만 돌아갑니다.
- **입력 키**: 
  - 키보드 **`R` 키** (KeyDownEvent)
  - 마우스 **우클릭** (포인터 우클릭 이벤트 또는 대체 터치 이벤트)
- **동작 흐름**:
  1. 위 조건에서 입력 이벤트 감지 시, `self.dragState.itemEntity` 안에 있는 컴포넌트 메서드 `Item:Rotate90Degrees()`를 호출합니다.
  2. 회전이 렌더러에 즉시 반영되므로, 곧바로 `HighlightPlacement` (초록/빨강 불빛)를 현재 마우스 위치 기준으로 **다시 갱신 호출**하여 바뀐 배치 윤곽선을 유저에게 실시간으로 보여줍니다.

### 5-2. 회전 알고리즘 (Item.mlua)
- **신규 메서드: `Rotate90Degrees()`**
  1. 기하학적 2차원 배열의 **시계 방향 90도 회전 공식**을 적용합니다.
     - 기존 행렬의 `(r, c)` 요소는 새로운 행렬의 `(c, 최대행 - 1 - r)` 요소로 이동합니다.
  2. `self.shapeMatrix` 변수 자체를 새롭게 연산된 회전 행렬로 덮어씌웁니다.
  3. 최대 너비와 높이(`self.width`, `self.height`) 값을 **서로 교환(Swap)** 합니다. (예: 2x3 직사각형 배열이었다면 3x2 배열이 되므로)
  4. **시각적 회전**: `self.Entity.UITransformComponent`의 `zAngle` (또는 UI 회전 프로퍼티) 값에 `-90`(또는 `+90`, MSW 좌표계에 맞게)을 누적하여 그래픽 렌더러도 함께 돌아가게 만듭니다.

## 6. Codex 작업 지시 요약

1. **`Data Table`**: 아이템 테이블에 `shapeMask` 필드 도입 및 `T`자나 `ㄱ`자 더미 데이터 입력. (문자열 기반 패턴 파싱)
2. **`Item.mlua`**: `GetShapeCells()`가 이제 네모난 넓이가 아니라 `shapeMask`의 "1"인 블록 좌표들의 배열 모음만 정확하게 반환하도록 재설계.
3. **`Inventory.mlua`, `InventoryUI.mlua`**: `CanPlace`, `PlaceItem`, `HighlightPlacement`의 모든 이중 for문(직사각형 방식)을 `shapeCells` 배열 순회 방식(비정형 방식)으로 교체 및 인자 변경.
