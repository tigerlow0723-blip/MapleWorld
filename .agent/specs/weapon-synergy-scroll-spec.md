# 목표 (Objective)

인벤토리에서 **무기(Weapon) 장비의 좌(Left), 우(Right)에 인접하여 배치된 공격력 주문서**가 무기 공격력에 시너지(Synergy)를 제공하도록 구현합니다.
해당 주문서는 **하급(ID 6: 1~20), 중급(ID 7: 1~40), 상급(ID 8: 1~100)** 3종이며, 아이템 획득/구매 시점에 랜덤하게 시너지 공격력 수치가 영구적으로 부여되어야 합니다. (위치를 바꿔서 난수를 리롤(Reroll)하는 꼼수 방지)

# 구현 상세 목록 (Implementation Details)

### 1. Item.mlua 파일 (시너지 공격력 속성 추가 및 난수 롤링)
**목적:** 주문서 아이템 엔티티가 생성될 때(보상 출현 또는 상점 구매) 고정된 랜덤 시너지 수치를 부여함.
- **로직 구현 흐름:**
  1. `Item` 컴포넌트 프로퍼티에 `bonusSynergyAtk` (integer, 기본값 0)를 추가합니다.
  2. `InitFromTable(integer id)` 함수 내에서 데이터 초기화가 끝난 후, 자신의 `id`가 시너지 주문서 중 하나인지 판별합니다.
  3. 시너지 주문서인 경우 `self.bonusSynergyAtk`가 0일 때만 (이미 값이 있으면 무시) 난수를 롤링하여 할당합니다.
     - `id == 6` (하급 무기 공격력 주문서): `math.random(1, 20)`
     - `id == 7` (중급 무기 공격력 주문서): `math.random(1, 40)`
     - `id == 8` (상급 무기 공격력 주문서): `math.random(1, 100)`
  - *참고사항:* 현재 인벤토리/상점 로직상 드래그 앤 드롭 시 기존의 엔티티 위치만 이동하므로 `InitFromTable`이 다시 호출되지 않아 값이 한 번 정해지면 영구 유지됩니다. (단, 추후 세이브/로드 기능 구현 시 해당 수치의 직렬화가 필요함을 유념해야 합니다.)

### 2. PlayerManager.mlua 파일 (좌우 인접 시너지 계산)
**목적:** `CalcFinalATK`에서 플레이어의 총 공격력을 계산할 때, 무기 컴포넌트 좌/우에 붙은 시너지 주문서를 스캔하여 최종 공격력에 합산함.
- **로직 구현 흐름:**
  1. `CalcFinalATK()` 함수 내의 `itemAtk` 계산 루프 근처에서 인벤토리 그리드 기반 시너지를 계산하는 로직을 추가합니다.
  2. `self.inventoryComp`를 통해 현재 그리드에 배치된 배치 정보(`placedItems`)를 순회합니다.
  3. 해당 아이템 데이터가 `"weapon"` 타입인 경우, 무기가 차지하는 공간(`shapeCells`)의 모든 칸(row, col)에 대해 **바로 왼쪽(col-1)**과 **바로 오른쪽(col+1)** 칸의 셀 정보(`self.inventoryComp.grid[r][c]`)를 확인합니다.
  4. 좌/우 셀에 존재하는 `itemId`가 시너지 주문서(6, 7, 8)에 해당하는지 확인합니다.
  5. 일치하는 주문서 코드가 발견되면 중복 체크를 하여 하나의 주문서가 동일한 무기에 여러 번 적용되지 않도록 방어(Set 활용)한 뒤, 해당 주문서 UI 컴포넌트(Entity.Item)를 찾아 `bonusSynergyAtk` 값을 `itemAtk` 에 합산합니다.
  6. 주문서 UI 컴포넌트를 찾는 헬퍼 함수 (`GetItemComponentById(itemId)`) 를 구성하여, `InventoryPanel` 하위 엔티티들 혹은 전체 자식들을 통해 해당 아이템 엔티티를 검색합니다. (현재 MSW 구조상 `itemId`가 그리드 키로 사용되고 있어 고유 식별 역할을 수행함)

### 3. ItemTooltipUI.mlua 파일 (시너지 수치 표시)
**목적:** 툴팁 패널에 해당 주문서가 가진 고유 랜덤 시너지 수치를 표시함.
- **로직 구현 흐름:**
  1. 주문서 아이템 툴팁을 띄울 때, `FindTooltipItemComponent(itemId)` 함수 등을 이용해 현재 타겟이 되는 아이템 컴포넌트에 접근합니다.
  2. 타겟 아이템이 주문서(ID 6, 7, 8)라면 툴팁 텍스트에 "Weapon Synergy: +[bonusSynergyAtk] (Left/Right)" 내용을 눈에 띄게 추가하여 표시해 줍니다.
  3. (옵션) MSW Maker의 `TooltipPanel` 하위에 UI Text 엔티티(`TxtSynergy`)를 새로 만들고 연결하거나, 기존의 설명(Desc) UI를 합쳐서 렌더링하도록 뷰 로직을 수정합니다.

# 코딩 가이드라인 (Coding Guidelines) / 주의사항
- MSW의 `pcall` 등 방어적인 코드가 필요한 곳이 있다면 적절하게 에러 핸들링을 구성합니다.
- 영문 주석 및 텍스트 레이블은 코드 컨벤션에 맞춰야 합니다.
- **다시 강조하지만, 본 명세서에는 코드를 직접 작성하지 마세요.** Codex가 위 규격을 읽고 스크립트에 반영할 수 있도록 가이드를 제공하는 역할입니다.
- 그리드 기반 좌/우 체크 시, 그리드 인덱스 범위(`1~maxWidth`, `1~maxHeight`)를 이탈하지 않도록 `Bounds Check`를 철저히 해야 합니다.
