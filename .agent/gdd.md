## 1. 인벤토리

### 1-1. 인벤토리 규격
- 최대 크기: 6×6 (36칸)
- 초기 크기: 3×3 (9칸), 6×6 그리드 중앙에 배치
- 셀 크기: 60×60px

### 1-2. 그리드 셀 상태

| 상태 | 값 | 색상 | 설명 |
|------|---|------|------|
| 비활성 | 0 | 어두운 회색 | 접근 불가 |
| 확장 가능 | -1 | 평소 숨김 / 드래그 시 반투명 | 활성 칸 인접, 확장 아이템으로 활성화 가능 |
| 빈칸 | 1 | 파란색 | 아이템 배치 가능 |
| 점유 | 2+ | 보라색 | 아이템이 배치됨 |

### 1-3. 드래그 & 드롭
- 아이템을 터치한 상태를 유지(홀드)하면 드래그 시작
- 드래그 중 아이템의 중심점이 커서/손가락 위치에 고정됨
- 하이라이트: 아이템 중심에서 크기를 역산하여 좌상단 기준 셀을 결정하고, 해당 셀부터 w×h만큼 하이라이트 표시
  - 배치 가능 → 초록색 / 배치 불가 → 빨간색
- 놓기 → 배치 성공 시 해당 셀에 스냅 / 실패 시 최초 생성 위치로 복귀

### 1-4. 인벤토리 확장
- 확장 아이템(1×1) 드래그 시에만 확장 가능 셀 표시
- 확장 가능 셀에 놓으면 → 해당 칸 활성화 + 확장 아이템 소비
- 활성화된 칸 주변에 새로운 확장 가능 셀 자동 생성

### 1-5. 아이템 회전 *(미구현)*
- 드래그 중 R키 또는 버튼으로 90도 회전
- 1×2 ↔ 2×1, 1×3 ↔ 3×1 등 width와 height 교환

## 2. 아이템

### 2-1. 아이템 분류

| 타입 | 설명 | 예시 |
|------|------|------|
| weapon | 공격력 부여 | 단검(1×2) |
| armor | 방어력 부여 | 나무방패(2×2) |
| consumable | 사용 시 효과 | 포션(1×1) |
| expand | 인벤토리 확장 | 확장카드(1×1) |

### 2-2. 아이템 속성
- id: 고유 번호
- name: 이름
- type: 분류
- width × height: 차지하는 칸 수
- atk / def / hp: 스탯
- spriteRUID: 이미지 리소스 ID
- desc: 설명

### 2-3. 아이템 배치 규칙
- 활성 셀에만 배치 가능
- 아이템끼리 겹침 불가
- 배치 실패 시 최초 생성 위치로 복귀
- 배치된 아이템은 다시 드래그하여 재배치 가능

### 2-4. 아이템 데이터 (ItemTable.csv)

| id | name | type | width | height | atk | def | hp | desc | spriteRUID |
|----|------|------|-------|--------|-----|-----|----|------|-----------|
| 1 | 단검 | weapon | 1 | 2 | 5 | 0 | 0 | 기본 단검 | (RUID) |
| 2 | 나무 방패 | armor | 2 | 2 | 0 | 3 | 0 | 기본 방패 | (RUID) |
| 3 | 포션 | consumable | 1 | 1 | 0 | 0 | 10 | HP 회복 포션 | (RUID) |
| 10 | 인벤토리 확장 | expand | 1 | 1 | 0 | 0 | 0 | 칸 1개 활성화 | (RUID) |

### 2-5. 아이템 나열 (진열)

아이템 지급 시(게임 시작, 보상 등) 아이템을 보여주는 영역.
플레이어는 이 영역에서 아이템을 드래그하여 인벤토리에 배치한다.

#### 지급 수량
- 최소 3개 ~ 최대 7개

#### 나열 규칙
- 진열 영역은 인벤토리 **좌측**에 **세로로** 배치
- **세로 1줄 기반**, 좌우 물결(지그재그) 형태
- 홀수 번째 아이템은 **왼쪽**, 짝수 번째 아이템은 **오른쪽** (좌우 오프셋)
- 아이템 수에 따라 세로 중앙 정렬, 간격 균등 배분
- 아이템 크기가 다르더라도 진열 시에는 **고정 크기 슬롯**(예: 80×80px)에 축소 표시

배치 예시 (5개):
```
  [1]
      [2]
  [3]
      [4]
  [5]
```

#### 동작
- 진열 영역의 아이템을 드래그 → 인벤토리에 배치
- 배치 실패 시 진열 영역 원래 자리로 복귀 (spawnPosition)
- 모든 아이템 배치 완료 시 다음 단계로 진행 (또는 배치 건너뛰기 가능 여부 — 추후 결정)
